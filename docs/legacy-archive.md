# Legacy Server Repository

После стабилизации монорепо старый `evc-team-relay` repo должен перейти в legacy режим.

## Что делать со старым repo

1. Остановить внесение новых продуктовых изменений в старый repo.
2. Обновить его `README`, чтобы он указывал на `WoodGear1/evc-team-relay-selfhosted` как canonical monorepo.
3. Отключить старые release workflow в legacy repo.
4. Перевести repo в read-only режим на время стабилизации.
5. Архивировать repo только после хотя бы одного успешного release cycle из монорепо.
6. Убрать любые инструкции, которые ещё ведут на старый `GHCR` namespace или старый deploy contract.

## Что нельзя делать

- Нельзя архивировать старый repo до подтверждённого рабочего deploy через `GHCR`.
- Нельзя держать два активных release pipeline для одних и тех же сервисов.
- Нельзя продолжать вести прод-критичные патчи параллельно в двух репозиториях.
- Нельзя оставлять старые инструкции, из которых можно случайно задеплоить `entire-vc/evc-team-relay`.
